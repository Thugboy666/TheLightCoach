# Schema concettuale (Core)

## 1) Osservazioni (input)
- Testo chat incollato
- Trascrizione voce (solo con consenso esplicito)
- Note utente (stato fisico/emotivo, contesto, eventi)

## 2) Segnali (features)
Esempi:
- pressione/urgenza ("rispondi ora", chiamate a raffica)
- colpa ("mi fai male", "sei tu il problema")
- ambiguità ("ti auguro il meglio" + richieste affettive senza scelta)
- triangolazione (terze persone come leva)
- controllo soft (micro-test, like tattici, puntini, "dove sei")

## 3) Pattern (profili)
Ogni profilo definisce:
- Nome
- Descrizione
- Segnali tipici (lista)
- Trigger comuni
- Rischi per l’utente
- Contromisure
- Frasi consigliate (micro-scripts)

## 4) Scoring (rischio)
Scala consigliata 0–10:
- 0–2: sano/gestibile
- 3–5: attenzione (confini)
- 6–8: dinamica tossica attiva (protezione)
- 9–10: rischio alto (disingaggio + supporto esterno)

## 5) Output (azioni)
- 1 frase (minima) oppure nessuna risposta
- Confine chiaro + conseguenza
- Piano 24–72 ore (micro-routine)
- “Crisis guardrail” se compaiono segnali di self-harm

## 6) Moduli
Un modulo è un layer che cambia:
- obiettivo (es. vendita vs relazione)
- stile comunicativo
- metriche di successo
ma usa lo stesso motore di:
- segnali → pattern → azioni
